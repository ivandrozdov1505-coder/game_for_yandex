# Выживи урок

«Выживи урок» — браузерная 2D (DOM-based) игра для формата Яндекс Игр.
Игрок управляет школьником на уроке: нужно дожить до звонка (или продержаться как можно дольше), реагируя на случайные события и удерживая параметры в допустимых пределах.

## Идея
Каждую секунду параметры героя меняются естественным образом (сонливость, стресс, подозрение, знания), периодически всплывают события, и игрок выбирает одно из действий с риском и выгодой. Игра поощряет баланс между безопасными и рискованными решениями.

## Основные механики
- **Параметры:**
  - сонливость
  - подозрение учителя
  - стресс
  - знания
  - очки
- **Случайные события (с разной опасностью):**
  - учитель смотрит в твою сторону
  - тебя спрашивают
  - сосед просит помочь
  - сообщение на телефоне
  - хочется спать
  - проверка тетрадей
  - мини-контрольная
  - вызов к доске
- **Действия игрока:**
  - слушать
  - делать вид, что работаешь
  - сидеть в телефоне
  - списывать
  - отвечать
  - игнорировать
  - отмазаться
  - попроситься выйти
- **Проигрыш:**
  - критическое подозрение
  - критическая сонливость
  - критический стресс
  - провал важного события
- **Сложность:** плавно растёт по мере хода урока.

## Режимы
1. **Обычный урок** — базовый режим, доступен сразу.
2. **Контрольная** — более стрессовый режим, открывается по рекорду.
3. **Бесконечный режим** — усложнённый выживательный режим, открывается позже.

## Сохранения (localStorage)
Ключ: `survive-lesson-save-v1`

Сохраняется:
- лучший счёт
- лучший счёт в режиме «Контрольная»
- открытые режимы

## Стек и архитектура
- HTML + CSS + JavaScript (без тяжёлых библиотек)
- Полностью статический проект, без серверной части

Структура:

```text
.
├── index.html
├── package.json
├── src
│   ├── js
│   │   ├── config.js      # версии, режимы, параметры, действия
│   │   ├── events.js      # таблица событий
│   │   ├── state.js       # централизованное состояние
│   │   ├── logic.js       # расчёты, тики, последствия действий
│   │   ├── ui.js          # отрисовка и обработчики UI
│   │   ├── storage.js     # localStorage
│   │   └── main.js        # точка входа и склейка модулей
│   └── styles
│       └── styles.css
└── README.md
```

## Запуск
### Вариант 1: через Python
```bash
python3 -m http.server 8080
```
Открыть: `http://localhost:8080`

### Вариант 2: через npm script
```bash
npm run start
```

## MVP-границы (что сделано в первую очередь)
- стартовый экран
- полностью рабочий режим «Обычный урок»
- живая система параметров
- случайные события
- действия игрока с последствиями
- условия поражения
- экран результата
- сохранение рекорда

После MVP добавлены: второй режим, бесконечный режим, полировка UI.

## Подготовка под Яндекс Игры
- проект запускается как статический сайт
- нет обязательной авторизации
- нет серверной части
- модульная структура позволяет позже добавить SDK без переписывания ядра логики
