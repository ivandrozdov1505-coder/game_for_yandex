# Выживи урок

**Выживи урок** — браузерная DOM-based игра для Yandex Games про выживание на уроке. Игрок держит баланс между сонливостью, стрессом, подозрением учителя и знаниями, реагирует на события класса и пытается дожить до звонка.

## Обновление 1.4.0

В версии **1.4.0** игра получила три крупных улучшения:

- Добавлено короткое обучение на 5 шагов.
- Обучение автоматически показывается при первом запуске, умеет продолжаться на первом реальном забеге, поддерживает `Пропустить` и доступно повторно из меню.
- Экран поражения теперь разбирает реальную причину проигрыша: главный фактор и вторичные риски на основе финальных параметров.
- Все эмодзи убраны из интерфейса и текстов. Вместо них используется внутренний SVG-спрайт и единый векторный стиль.
- Обновлены стартовый экран, игровая сцена, карточки параметров, события, кнопки действий и result screen.

## Что есть в игре

- Три режима:
  - **Обычный урок** — нужно дожить до звонка.
  - **Контрольная** — более жёсткий сценарий с высоким давлением.
  - **Бесконечный режим** — игра на рекорд без финального звонка.
- Центральные параметры:
  - `sleepiness`
  - `stress`
  - `suspicion`
  - `knowledge`
  - `score`
- Случайные события урока и набор действий с разным риском и пользой.
- Поражение по критическим параметрам или по провалу важного события.
- Локальные сохранения с валидацией и миграцией.
- Поддержка облачного сохранения через Yandex Player API, если игрок авторизован.
- Системная пауза, interstitial и rewarded ads через SDK-обёртки.

## Обучение

Первый запуск теперь устроен так:

1. В меню появляется первый шаг обучения с целью игры.
2. После нажатия `Далее` игрок сам запускает забег.
3. На первом реальном уроке игра ставится на tutorial-паузу и быстро объясняет HUD, события, действия и условия поражения.
4. После завершения или пропуска обучение больше не показывается автоматически.

Кнопка `Обучение` в меню запускает безопасный preview-урок без влияния на статистику, рекорды и прогресс.

## Экран поражения

Result screen теперь показывает:

- основной фактор поражения;
- реальные значения параметров;
- вторичные факторы, если они были опасно высокими;
- отдельную подсказку про проваленное важное событие, если оно добило забег.

Это помогает понять, что именно игрок делал не так в конце раунда.

## Запуск локально

### Вариант 1

```bash
python3 -m http.server 8080
```

Открыть `http://localhost:8080`.

### Вариант 2

```bash
npm run start
```

Проверка синтаксиса:

```bash
npm run check
```

## Архитектура

- `logic.js`, `config.js`, `events.js` — игровая логика и баланс.
- `state.js`, `core/game-states.js` — состояние забега и состояния приложения.
- `ui.js`, `styles.css`, `index.html` — рендер экранов, tutorial overlay, SVG-подача.
- `tutorial.js` — декларативные шаги обучения.
- `result-analysis.js` — вычисление причин поражения по финальному состоянию.
- `storage.js` и `sdk/storage-bridge.js` — локальные и облачные сохранения.

## Сохранения

Хранятся:

- рекорды;
- открытые режимы;
- агрегированная статистика;
- звуковые настройки;
- флаг `tutorialShown`.

Ключ localStorage сохранён прежним: `survive-lesson-save-v2`.

## Ограничения и future work

- Звук пока остаётся сервисным слоем без полноценных audio assets.
- В игре нет отдельной meta-progression поверх разблокировки режимов.
- Возможные следующие улучшения:
  - больше событий и режимов;
  - ежедневные задания;
  - достижения;
  - дополнительные визуальные реакции сцены;
  - более глубокая аналитика по забегам.
