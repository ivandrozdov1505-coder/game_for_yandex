# Выживи урок

**Выживи урок** — браузерная 2D (DOM-based) survival/reaction/management игра, подготовленная под публикацию в Яндекс Играх.

Игрок — школьник, который должен дожить до конца урока, управляя риском по параметрам: сонливость, стресс, подозрение учителя, знания и очки.

## Что реализовано

- Рабочий игровой цикл (menu → playing → result).
- Режимы:
  - **Обычный урок** (MVP, доступен сразу).
  - **Контрольная** (разблокируется по очкам).
  - **Бесконечный режим** (разблокируется позже).
- Все обязательные события и действия из ТЗ.
- Проигрыш по критическим параметрам и по провалу важного события.
- Пауза вручную и системная пауза (платформа/реклама).
- Локальные сохранения с валидацией и миграцией версии.
- Базовый мост к облачному сохранению через Player API (если игрок авторизован).
- Интеграция Yandex Games SDK с безопасным fallback в локальной разработке.

## Механики

### Параметры
- Сонливость (критична на 100)
- Подозрение учителя (критично на 100)
- Стресс (критичен на 100)
- Знания (помогают на проверках)
- Очки (прогресс и разблокировки)

### События
- Учитель смотрит в твою сторону
- Тебя спрашивают
- Сосед просит помочь
- Сообщение на телефоне
- Клонит в сон
- Проверка тетрадей
- Мини-контрольная
- Вызов к доске

### Действия
- Слушать
- Делать вид, что работаешь
- Сидеть в телефоне
- Списывать
- Отвечать
- Игнорировать
- Отмазаться
- Попроситься выйти

Каждое действие имеет риск/пользу и влияет на параметры.

## Архитектура

### Слои
1. **Игровая логика** (`logic.js`, `events.js`, `config.js`) — чистые правила, баланс, события.
2. **Состояние** (`state.js`, `core/game-states.js`) — run-state и app-state.
3. **UI** (`ui.js`, `styles.css`, `index.html`) — отрисовка, без прямых вызовов SDK.
4. **SDK-слой** (`src/js/sdk/*`) — вся работа с платформой через обёртки.
5. **Сохранения** (`storage.js`, `sdk/storage-bridge.js`) — локально всегда, облако опционально.

### Состояния приложения
- `boot`
- `loading`
- `menu`
- `playing`
- `paused`
- `result`
- `ad`
- `system_pause`

Именно от этой машины состояний зависит вызов GameplayAPI start/stop.

## Интеграция Yandex Games SDK

### Инициализация
- Инициализация выполняется в `src/js/sdk/init.js` через `YaGames.init()`.
- Есть защита от повторной инициализации (`initPromise`).
- В локальной среде (без SDK) используется безопасный fallback, игра остаётся полностью играбельной.

### Где вызывается `LoadingAPI.ready()`
- В `bootstrap()` (`src/js/main.js`) после:
  - init SDK,
  - загрузки/слияния сохранений,
  - подготовки UI меню,
  - готовности первого интерактивного экрана.

### Где и как используются `GameplayAPI.start()/stop()`
- В `src/js/sdk/gameplay.js` — централизованный синк с app-state.
- В `setAppState()` (`main.js`) при каждом переходе состояния.
- Start только в `playing`, stop во всех не-геймплейных состояниях.

### `game_api_pause` / `game_api_resume`
- Подписка в `src/js/sdk/events.js`.
- Обработка в `main.js`:
  - pause: стоп лупа, заморозка игры, переход в `system_pause`, пауза звука.
  - resume: возврат в `playing` только когда корректно (без дублей лупа).

## Реклама

Реализована только через SDK-обёртки:
- `showInterstitial()`
- `showRewarded()`

Модуль: `src/js/sdk/ads.js`.

Поведение:
- Fullscreen interstitial показывается после завершения раунда.
- Rewarded — только по явной кнопке «Второй шанс за рекламу».
- Награда выдаётся только если пришёл `onRewarded` callback.
- Во время рекламы игра и звук ставятся на паузу, затем корректно возвращаются.

## Сохранения

### Локальные
Модуль `src/js/storage.js`, ключ: `survive-lesson-save-v2`.

Сохраняется:
- best score,
- best test score,
- unlocked modes,
- aggregate stats (runs/wins/losses),
- sound settings.

Есть:
- валидация структуры,
- миграция версии,
- защита от битого JSON.

### Облачные (опционально)
`src/js/sdk/storage-bridge.js`:
- игра работает без логина и без облака,
- если Player API доступен и пользователь авторизован — данные читаются/пишутся в облако,
- локальные сохранения остаются источником работоспособности в любом случае.
